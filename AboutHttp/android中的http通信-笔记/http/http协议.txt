http协议是什么？
  超文本传输协议hypertexttransferpeotocol，定义了浏览器怎样向万维网服务器请求万维网文档，以及服务器怎样把文档传给浏览器
  从层次的角度看，http是面向transaction-oriented应用层协议，它是万维网上能够可靠地交换文件的重要基础

***什么是URL？***
统一资源定位符（UniformResourceLocator）也叫网页地址，是因特网上标准的资源的地址
url用来标识网络上唯一的网址信息
URL的格式由下列三部分组成：
第一部分：协议(或服务方式)
第二部分：存有该资源的主机IP地址(有时也包括端口号)
第三部分：主机资源的具体地址，如目录和文件名等

第一部分和第二部分之间用“：//”符号隔开，第二部分和第三部分用“/”符号隔开
第一部分和第二部分是不可缺少的，第三部分有时可以省略，
https 是加密的协议
http://www.imooc.com/index.jsp
http就是协议，接下来是域名，它的背后通过dns服务器去解析相应的主机地址信息  
（dns服务器是计算机域名系统(Domain Name System 或Domain Name Service)，是一种组织成 域层次结构 的计算机和网络服务命名系统，
它是由解析器和域名服务器组成的，用于TCP/IP网络）最后是具体的资源地址信息



有了URL之后就要跟服务器建立连接：
要先创建一个tcp连接，tcp/ip建立连接的时候，要经过三次握手才能连接上

***tcp/ip的三次握手***
两个概念：
  1、SYN（syncronous）：tcp/ip建立连接时使用的握手信号。
          客户端先向服务器发送SYN信息，表示要跟服务器建立tcp/ip连接；
          服务器使用SYN+ACK应答表示接收到了这个消息，可以建立连接；
          客户机再以ACK响应，表示客户端也可以建立连接
  2、ACK（acknowledgement）：确认字符，接收站发送给发送站的一种传输类控制字符，表示发来的数据已确认接收无误

http请求头信息  。。。URL、请求方式（get）、状态码（200表示成功）、编码信息、压缩信息、接收的语言、连接(常连接--keep alive)、kookie、主机地址等

http的请求方式：
get  会把所有的参数根据URL一起传送到服务器***
post 会把所有请求数据封装成http请求的实体发送过去，在访问页面的时候就不会发现请求的参数是什么***
head请求获取参数的响应消息报头
put请求服务器存储一个资源，并用Request-URL作为其标识
delete请求服务器删除Request-URL所标识的资源
trace请求服务器回送收到的请求信息，主要用于测试或诊断
connect保留将来使用
options请求查询服务器的性能，或者查询与资源相关的选项

发送完请求之后服务器的应答叫做响应头
响应头的响应码信息：
200 OK//客户求成功
400 Bad Request 客户端请求有语法错误
401 Unauthorized 请求未经授权，这个状态码必须跟WWW-Authenticate报头域一起使用
403 Forbidden 服务器收到请求，但是拒绝提供服务
404 Not Found 请求资源不存在 如输入了错误的URL
500 Internal Server Error 服务器发生了不可预测的错误
503 Server Unvaliable 服务器当前不能处理客户端的请求，一段时间后可能恢复正常
